我建议的优化方案

一、每节固定成“三层代码”结构

每一节都尽量有这三类文件：

demo.py 或 examples.py
完整可运行
对应文档主讲内容
用来“先跑一遍，建立感觉”
exercises.py
保留空白题
只承载练习，不承担教学主线
solution.py 或 reference.py
练习参考答案
你可以先不看，卡住再看
这样你会很清楚：

文档看不懂时，先跑 demo
想练习时，做 exercises
卡住时，对照 solution
如果你不想公开标准答案，也可以只在本地保留 reference.py。

二、文档不要只讲知识点，要讲“操作顺序”

每一节文档开头都建议固定加一个“本节怎么学”区块，写得非常直接：

先运行哪个文件
再阅读哪几个小节
哪些必须手写
哪些可以先看答案再回写
本节学完要能做到什么
这会极大降低你的启动成本。

现在很多文档内容本身不少，但“学法说明”不够强，所以读起来像资料，不像路线。

三、文档和代码做显式映射，不要靠你自己脑补

建议每节都加一个小表格：

文档章节 3.1 变量
对应代码 basic.py 的哪一段 / demo.py 的哪个函数
对应练习 exercises.py 的哪个题
你说的“代码和文档一一对应”，我认同，而且这是关键。否则文档是文档，练习是练习，脑中不会形成结构。

四、exercises.py 不要承担“首次理解”职责

这是你现在最难受的地方。空白练习适合巩固，不适合入门理解。

所以前期 📌 章节里，exercises.py 应该排在第三位，不是第一位。顺序应该是：

读文档中的最小例子
跑完整可运行示例
自己手写 exercises.py
对照参考答案修正
你现在相当于跳过了第 2 步，直接面对空白题，阻力太大。

五、前期章节建议“少而稳”，不要一开始铺太多练习

对你当前状态，我不建议每节都设计很多开放题。前 1-4 节优先保证：

每节 1 个完整示例
每节 3-5 个非常明确的小练习
每节 1 个综合小题
不要一上来做太多“可扩展练习”，否则会继续稀释主线。

### 示例

```
### 2.1 列表（List）📌

**一句话解释**：列表就是一个能装多个东西的容器，像数组但更灵活。

**为什么需要**：你想存 5 个学生的成绩，用 5 个变量太麻烦，用一个列表就够了。

**怎么用**：
```python
# 创建列表
scores = [85, 92, 78]

# 取值（从 0 开始）
print(scores[0])  # 输出：85

# 添加元素
scores.append(90)  # 变成 [85, 92, 78, 90]

# 获取长度
print(len(scores))  # 输出：4
```

**常见坑**：
- ⚠️ 索引越界：`scores[10]` 会报错，Python 不会像 JS 那样返回 undefined
- ⚠️ 列表切片：`scores[1:3]` 不包含第 3 个元素，是 `[92, 78]` 不是 `[92, 78, 90]`

**AI 应用场景**：存储对话历史、检索结果、Prompt 模板列表
```
