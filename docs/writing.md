# 文档与代码质量规范

> 定位：本文件是当前仓库内文档、课程正文、配套代码 README 和示例代码质量要求的唯一真源。

## 1. 目录职责

### `docs/`

只放仓库治理类文档，包括：

- 写作规范
- 课程和章节模板
- 质量检查清单
- 协作约定
- skill 或 agent 相关说明

### `course/`

只放课程正文，包括：

- 阶段大纲
- 阶段 README
- 章节学习文档
- 项目规划文档
- 课程相关方法论文档

### `source/`

只放课程配套代码和资源，包括：

- 示例代码
- 练习代码
- 测试
- prompt、数据集、评估集
- 章节级或项目级代码 README

## 2. 核心原则

### 一节一闭环

每一章、每一节、每一个小项目都必须形成最小闭环：

1. 明确目标
2. 清晰入口
3. 对应代码
4. 学习或实施顺序
5. 完成标准

### 先入口，再扩展

先提供最小可运行入口，再补练习、扩展案例、对照实现和工程化封装。

不应把半成品练习、零散代码片段或缺少运行说明的项目骨架作为第一入口。

### 显式映射

课程文档必须说明：

- 当前内容对应哪些代码文件
- 哪个文件是第一入口
- 哪些文件是练习、对照或扩展
- 哪些测试或命令用于判断完成

### 文档服务学习路径

文档不是资料堆放处，而是学习路线。每篇课程文档至少要回答：

- 为什么学
- 学到什么程度
- 先做什么
- 容易错在哪里
- 学完如何判断自己达标

### 代码服务理解与实践

示例代码优先服务理解和动手验证。不要为了“看起来工程化”提前堆复杂目录和抽象。

章节级内容先保证核心链路正确、可跑、可解释；项目级内容再逐步补配置、分层、日志、测试和接口。

## 3. 课程文档标准

章节文档不强制标题完全一致，但必须覆盖这些职责：

- 本章目标
- 本章在整体路径中的作用
- 学习前提和边界
- 推荐学习顺序
- 第一阅读入口或第一运行入口
- 核心概念、流程或抽象
- 代码映射
- 练习或任务说明
- 完成标准

目标写法应使用可验证动词，例如：

- 能运行
- 能解释
- 能实现
- 能区分
- 能修改

避免只写“了解”“熟悉”“掌握”但没有验证方式。

## 4. 代码 README 标准

`source/` 下的章节或项目 README 必须说明：

- 当前代码对应哪篇课程文档
- 推荐先运行哪个文件
- 需要哪些依赖和环境变量
- 最小运行命令是什么
- 测试命令是什么
- 当前已经完成什么
- 当前不支持什么

代码 README 不负责重复讲完整理论，应优先说明“怎么跑、怎么看、怎么验证”。

## 5. 代码质量标准

示例代码应满足：

- 文件命名清楚，能看出学习顺序
- 主示例、练习、工具函数和测试职责分开
- 外部依赖有 `requirements.txt` 或清晰说明
- 涉及真实模型、数据库、向量库时提供 mock 或 fallback 路径
- 配置和密钥分离，真实密钥不能写入仓库
- 关键示例有最小测试或可重复验证命令

阶段代码复杂度应随学习目标增长：

- `01_python`：强调语法、练习和参考实现
- `02_llm`：强调 `.env.example`、provider 配置、CLI/API/service 边界
- `03_foundation`：强调抽象边界、native 与框架对照、最小项目骨架
- `04_rag`：强调文档处理、检索、生成、评估和坏案例闭环
- `05_agent`：强调工具、状态、路由、工作流和可观测性
- `06_application`：强调完整产品闭环、前后端联调、部署和演示

## 6. 路径与链接规范

所有指向仓库内文件的链接必须使用相对路径。

禁止在仓库文档中写本机绝对路径，例如：

```plain
/home/user/work/...
```

推荐：

```md
`course/04_rag/outline.md`
`source/04_rag/01_rag_basics/README.md`
```

目录引用规则：

- 课程正文引用课程正文：使用相对路径指向 `course/`
- 课程正文引用代码：使用相对路径指向 `source/`
- 代码 README 引用课程正文：使用相对路径指向 `course/`
- 根目录 README 引用任何文件：使用从根目录出发的相对路径

## 7. 完成定义

一章内容完成，至少满足：

- 课程文档有目标、边界、学习顺序、代码映射和完成标准
- 配套代码能按 README 跑通
- 关键路径有测试或可重复验证命令
- 文档和代码互相能找到
- 没有失效链接、旧目录引用或本机绝对路径

一个阶段完成，至少满足：

- 阶段 README 能说明学习顺序
- 阶段 outline 能说明整体目标和边界
- 每章都有独立闭环
- `course/` 和 `source/` 的映射清楚
- 后续阶段能直接复用该阶段产物

## 8. 评审清单

修改文档或代码后，至少检查：

- 是否仍有本机绝对路径
- 是否仍引用旧的课程目录路径
- 是否说明第一入口
- 是否说明运行命令
- 是否说明完成标准
- 是否新增了无法运行的示例
- 是否把课程正文误放进 `docs/`
- 是否把治理规范误放进 `course/`

## 9. 例外处理

允许临时不完整，但必须显式标记：

- 当前状态
- 缺什么
- 为什么暂时不补
- 后续补齐入口

不允许长期保留“看起来完整但实际不能跑、不能学、不能验证”的内容。
