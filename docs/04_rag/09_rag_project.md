# 09. 综合项目

## 1. 本章目标

- 能把前八章能力整合成一个完整 RAG 项目
- 能说明 `phase_9_project_integration` 和最终 `rag_lab` 的关系
- 能形成一个可运行、可评估、可继续扩展的 RAG 系统
- 能为后续 `06_application` 的业务项目准备知识底座

## 2. 本章在 04_rag 中的位置

本章是 `04_rag` 的收口章节。

前八章是分阶段学习快照，本章负责把这些能力整理成最终项目。

## 3. 学习前提

- 已完成 `phase_1 -> phase_8`
- 已能解释文档处理、向量化、检索、生成和评估之间的关系

## 4. 本章边界

本章做完整 RAG 项目，不做复杂前端、不做企业级后台、不做 Agent 工作流。

## 5. 学习顺序

1. 先回看 `rag_lab_design.md`
2. 再进入 `phase_9_project_integration`
3. 把前面 phase 的能力整合
4. 最后整理为最终 `source/04_rag/rag_lab/`

## 6. 核心知识点

- 项目整合
- API 收口
- README 与运行说明
- 最小测试
- 最小评估
- 当前边界和后续迁移

## 7. 对应代码

| 学习内容 | 对应代码 | 当前角色 |
|----------|----------|----------|
| 项目整合快照 | `source/04_rag/labs/phase_9_project_integration/` | 第九章学习入口 |
| 最终完整项目 | `source/04_rag/rag_lab/` | 最终参考实现 |
| 阶段任务说明 | `docs/04_rag/rag_lab_tasks.md` | 实施顺序 |
| 项目设计说明 | `docs/04_rag/rag_lab_design.md` | 架构约束 |

## 8. 实践任务

1. 整合文档处理、索引、检索、生成和评估
2. 补最小 API 或 CLI 入口
3. 补运行说明
4. 跑一次完整 Golden Set 回归
5. 写出项目未完成边界

## 9. 完成标准

- 最终 `rag_lab` 能作为完整参考项目存在
- 学习入口仍然是九章文档和 `labs/phase_*`
- 项目 README 能说明如何运行、如何评估、当前不支持什么
