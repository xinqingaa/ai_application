# AI 应用落地 学习大纲

> 目标：把前面的 `LLM / RAG / Agent` 能力真正落到一个可运行、可部署、可观测、可讲述的完整产品中，形成接近企业标准的 AI 应用开发能力

---

## 课程定位

- 本课程不是继续堆概念，而是把前面学到的能力串成一个真实系统。
- 本课程不建议一上来做“通用 LLMOps 平台”，因为那会把精力过早消耗在平台化、可视化编排、开放平台等重工程模块上。
- 更适合你的路线是：**先做强一个金融场景的 AI 应用平台，再逐步抽象出平台能力**。
- 本课程会吸收 `llmops.md` 里有价值的部分，例如：
  - 多模型接入
  - 记忆与知识库
  - 任务调度
  - 认证、权限、审计
  - 部署、监控、成本控制
- 但暂不把以下内容作为主线：
  - 可视化 Prompt 编排平台
  - 通用开放 API 平台
  - 多租户通用 LLMOps 后台
  - Kubernetes / 数据仓库 / SDK 体系
  - 数字人、PPT 生成、多模态插件大杂烩

## 主项目方向

### 金融 AI 助手平台

这个主项目不是单一聊天机器人，而是一个围绕金融信息、客服、知识库和合规场景的 AI 应用。

#### 推荐业务模块

1. **智能客服**
   - 券商 / 金融 App 常见问题问答
   - 交易规则、手续费、开户流程、产品说明
   - 支持来源引用、会话记忆、升级人工

2. **内部知识库**
   - 产品文档、FAQ、运营手册、接口说明、风控规则
   - 文档上传、切分、索引、检索、问答
   - 支持权限隔离与知识库管理

3. **AI 审核助手**
   - 客服回复审核
   - 运营文案审核
   - 金融内容风险提示
   - 输出审核原因和建议修正

4. **投研信息助手**
   - 新闻摘要
   - 财报 / 公告摘要
   - 个股事件时间线整理
   - 多来源信息聚合与风险提示

#### 不建议作为主卖点的内容

- 直接做“股票推荐”
- 直接做“自动交易决策”
- 直接做“收益预测神器”

原因：

- 难验证
- 难讲清楚依据
- 合规风险高
- 面试和包装时容易被追问到无法自圆其说

## 课程目标

完成本课程后，你应该具备以下能力：

1. **设计** 一个真实的 AI 应用系统架构，而不是只会写 demo
2. **实现** 前端、后端、RAG、Agent、数据库、缓存、异步任务之间的完整链路
3. **落地** 一个可运行的金融 AI 应用平台
4. **说明** 你的技术选型、系统边界、成本控制和安全策略
5. **具备** 后续继续平台化、产品化、工程化扩展的能力

## 技术栈建议

### 前端

- 优先使用你更有优势的技术栈：
  - Flutter
  - Web 前端（React / Next.js）

建议：

- 第一版可以优先做 `Web + 后台管理界面`
- 如果你想突出自己的差异化，再补一个 `Flutter 客户端`

### 后端

- FastAPI
- Pydantic
- SQLAlchemy / SQLModel
- Alembic

### 数据层

- MySQL 或 PostgreSQL
- Redis
- 对象存储：本地文件系统起步，后续可扩展 MinIO
- 向量库：
  - 起步：Chroma / pgvector

### AI 能力层

- OpenAI / Claude / 国内模型统一接入
- LangChain
- LangGraph
- RAG 检索链路
- Agent 工作流

### 工程化

- Docker Compose
- 日志系统
- Trace / 可观测性
- 基础监控和成本统计

### 5. 异步任务与任务调度

#### 知识点

   - 批量审核耗时
   - 报表生成耗时

2. **任务类型**
   - 文档索引构建
   - 数据清洗
   - 批量审核
   - 定时摘要任务

3. **设计重点**
   - 任务状态
   - 重试
   - 幂等
   - 失败告警

#### 实践练习

```text
设计一个文档处理任务流：
上传 -> 入库 -> 解析 -> 切分 -> 向量化 -> 建索引 -> 完成 / 失败

## 三、AI 应用核心链路

### 6. 聊天服务与流式体验

#### 知识点

1. **聊天主链路**
   - 创建会话
   - 保存消息
   - 调用模型
   - SSE / 流式返回

2. **用户体验**
   - 打字机效果
   - 中断回复
   - 重新生成
   - 历史会话回看

3. **前端优势发挥**
   - 你可以把这部分做到明显强于普通后端工程师

#### 实践练习

前端发送 -> SSE 流式展示 -> 保存历史 -> 支持重新生成
```

### 8. Agent 服务与工作流

#### 知识点

1. **先做一个可控 Agent**
   - 不追求全能
   - 只针对明确场景设计工具和流程

2. **推荐 Agent 场景**
   - 问题路由：普通问答 / 知识库问答 / 审核任务
   - 内容审核：判断风险类型并给出修改建议
   - 投研辅助：先检索，再总结，再输出风险提示

3. **LangGraph 应用**
   - 路由节点
   - 检索节点
   - 审核节点
   - 输出节点

#### 实践练习

```text
实现一个 LangGraph 工作流：
用户问题 -> 路由 -> RAG / 审核 / 普通聊天 -> 输出结果
```
